<objective>
Build the Vitest-based doctest harness that extracts tagged fenced code blocks from `README.md` AND every file under `docs/` and executes them against the package source. This is the DX-01 safeguard — from Wave 1 onward every tagged snippet is guaranteed to compile and run under a Noop reporter with Sentry mocked. Establishes the marker convention that plans 03-02 and 03-03 consume.

Purpose: Without this harness every doc edit is hope-driven. With it, CI rejects drift between docs and real behavior across the entire user-facing doc surface.
Output: Vitest alias config, extractor module, doctest suite covering README + docs/*.md, a seed fixture so Wave 1 already executes something, and a short contributor README explaining the marker.
</objective>
