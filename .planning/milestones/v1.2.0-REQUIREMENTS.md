# Requirements: @power-rent/try-catch (v1.2.0 — ARCHIVED)

**Defined:** 2026-01-31
**Shipped:** 2026-04-18
**Core Value:** A fluent Try API that never hides errors.

> Archived snapshot of REQUIREMENTS.md at v1.2.0 milestone close. All 14 v1 requirements satisfied.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Core Try API

- [x] **TRY-01**: User can wrap sync functions with `new Try(fn, ...args)` and execute via `.value()`, `.error()`, or `.unwrap()`
- [x] **TRY-02**: User can wrap async functions with the same fluent API and preserve type inference
- [x] **TRY-03**: User can provide a fallback via `.default(value)` and receive it on error
- [x] **TRY-04**: Error channel is type-safe across all execution paths

### Sentry Reporting

- [x] **SENT-01**: User can call `.report(message)` to capture the error via Sentry
- [x] **SENT-02**: User can attach tags and breadcrumbs to reported events via fluent chain
- [x] **SENT-03**: Errors are reported only when `.report()` is explicitly called, but breadcrumbs are always recorded when `.breadcrumbs()` is used
- [x] **SENT-04**: Breadcrumbs configuration is type-safe and matches README examples

### Entry Points

- [x] **ENTRY-01**: Node entry point available at `@power-rent/try-catch/node`
- [x] **ENTRY-02**: Browser entry point available at `@power-rent/try-catch/browser`
- [x] **ENTRY-03**: Next.js entry point available at `@power-rent/try-catch/nextjs`

### Diagnostics

- [x] **DIAG-01**: Non-Error throws are normalized into an Error before reporting
- [x] **DIAG-02**: Debug mode allows local error logging without Sentry

### Developer Experience

- [x] **DX-01**: Documentation and examples read like native English error handling

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Sentry Reporting

- **SENT-05**: Scoped metadata isolation per call to prevent cross-request bleed
- **SENT-06**: Zero runtime dependency when Sentry is unused

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| New runtimes (Deno/Bun/Cloudflare) | Focus on current environments for v1 |
| Result-style combinators (combine/all/partition) | Scope creep beyond focused Try API |
| Lint plugin for “must handle” semantics | Likely separate package later |
| Auto-report all errors by default | Creates Sentry noise; report is explicit |
