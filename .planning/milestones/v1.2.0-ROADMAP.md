## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Use return-type generics to preserve async/sync inference in Try | Avoid `Promise<unknown>` collapse for sync callers | ✓ Good |
| Treat `never` as non-promise in `IfPromise` | Avoid type collapse for callers that throw | ✓ Good |
| Support sync + async through same Try API | One API surface; lazy thenable installs only when needed | ✓ Good |
| Keep sync paths immediate, preserve Promise behavior for async | User expectation matches native error handling | ✓ Good |
| Normalize non-Error throws in `execute()` (not at adapter) | Single source of truth; adapters stay thin | ✓ Good |
| Unified breadcrumb path across all terminal branches | Eliminates surprise "breadcrumb only fires on report" | ✓ Good |
| Doctest harness over hand-maintained examples | Docs can never drift from `src/` | ✓ Good |
